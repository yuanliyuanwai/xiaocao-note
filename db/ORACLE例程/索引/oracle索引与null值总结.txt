总结：
    无论是单列唯一索引或复合唯一索引，对于可以为null的列或复合null值，Oracle不会为其存储索引值
    故在基于单列创建B树唯一索引或多列创建B树复合唯一索引的情形下
当列上允许为null值时
    where子句使用了is null的情形，其执行计划走全表扫描
    where子句使用了基于is not null的情形，起执行计划走索引扫描（索引访问扫描和全扫描）
当列上不允许为null值时，存在非null约束
    where子句使用了is null的情况，其执行计划走索引扫描
    where子句使用了is not null的情形，其执行计划也是走索引扫描
注：另外上述结论也适用于单列非唯一索引和复合非唯一索引（B树索引）
此在Oracle 10g R2(linux)下的情形，不同的优化器版本可能会有偏差。




如何让NULL值走索引
0.创建非空约束（当然没什么意思）
1.创建nvl函数索引（B树索引）
2.创建一个伪列的组合索引  
    create index i_new_t2_obj_id on t2(obj_id,-1) （其中obj_id可以为null）
总结
1.对于用于连接或经常被谓词使用到的列应尽可能避免NULL值属性，因为它容易导致索引失效
2.为需要使用NULL值的列添加缺省值(alter table tb modify(col default 'Y'))。
3.如果null值不可以避免也不能使用缺省值，应该考虑使用nvl函数索引，或者使用伪列来创建索引来提高查询性能
4.对于复合索引应保证索引中至少有一列不能null值，这是因为全部为null值时不被索引存储，以保证使用is null可以使用索引
5.当只需要检索某一个状态的数据(比如状态为0)可以使用decode(state,0, 0, null)利用索引不会存储null值的特性压缩索引
